[1] PRE-FLIGHT

What it does

Six terminals and four worktrees is not orchestration — it's tab-hunting. Pass turns supervision into a feed: every session reports in, and your keyboard does the rest.

PF-01

Summon anywhere

Choose ⌘⌘ (default) or ⌥Space in Settings to raise the panel over any app, on any Space — without stealing your editor's focus. Keep it floating on top, or dock it as a normal window beside your editor. Resizable; remembers its size.

⌘⌘ or ⌥space summon · ⌘⇧F float ⇄ dock

PF-02

Chat home

Every session as a card with its last response, newest attention on top. One omnibox pinned at the bottom: type to reply (injected safely — it refuses a bare shell), @ to jump, y/n to answer a pending permission.

@ jump · yn decide · ⏎ open

PF-03

A real terminal, embedded

Open a card and you're attached to its tmux session in a full interactive terminal — arrows, colors, permission menus, everything. Step back with ⌘[; hand off to Ghostty with ⌘⏎ when you want a real window.

⌘[ back · ⌘⏎ open in Ghostty

PF-04

Hook-driven attention

Claude Code hooks report permission prompts, questions, and completions the moment they happen — no polling, no screen-scraping guesswork. The session that needs you surfaces with exactly what it wants and how long it's been waiting, plus branch, worktree ⧉ badge, and agent glyph. Claude's numbered option menus mirror live into the card — drive them with plain ↑↓ ⏎ without opening the terminal.

notification · stop · prompt-submit hooks → 127.0.0.1 only

PF-05

tmux is the database

Sessions are plain tmux sessions: they survive Pass restarts, you can attach from any terminal, and existing pass-* sessions are adopted on launch. Branch, worktree, cwd, activity — all derived live from tmux + git, so nothing ever goes stale. Register a repo, or a parent folder scanned for repos; worktrees group under their main repo.

tmux attach -t pass-ivma — from anywhere

PF-06

Ambient signals

Native notifications for permission / input / finished, with the menu-bar badge as the always-on fallback that never asks for permission itself.

✳ 2 in the menu bar

PF-07

Share into a session

Use the macOS share sheet to send URLs, selected text, notes, images, and files directly into a live agent session, or start a new session in a saved project with the shared payload as the first prompt.

Share → Pass · live session or new session

PF-08

Mobile companion, planned

The planned mobile version turns your phone into a lightweight command deck: push notifications for blocked agents, approve or deny prompts, send quick replies, and share links or screenshots from mobile into the right desktop session.

iOS first · Android under evaluation · remote session handoff

[2] SYSTEM

How it fits together

Straight from the README. Hooks in, decisions out — and the terminal is always one keystroke away.

Claude Code (in tmux)  ──hooks(HTTP)──▶  HookServer (127.0.0.1:49817)
                                              │  normalize (ClaudeAdapter)
                                              ▼
   TmuxClient ◀── reconcile ──  SessionStore  ──▶  EventRouter ──▶  Inbox + Notifications
   (create/adopt/attach)        (git identity,        (state machine)          ▲
        ▲                        attention)                                    │
        └────────── ReplyInjector (bracketed paste / y·n) ◀── panel (SwiftUI) ──┘
Agent-agnostic coreAll agent knowledge lives in adapters. Claude Code today; the /hook/<agent> routes and per-agent glyphs for Codex & pi are already wired for M5.
Loopback onlyThe hook server binds 127.0.0.1 — no firewall prompt, no LAN exposure. Replies are injected with bracketed paste, and a bare shell prompt refuses injection.
Nothing to migratePass persists only a small project MRU list. Kill the app, reboot the Mac — your sessions are still in tmux, exactly where they were.
[3] KEYS

Hands stay home

The panel is built to be driven blind. Every path — summon, triage, decide, dive, dismiss — fits under your fingers.

⌥+space

Summon / dismiss the panel, over any app, any Space

@

Jump — fuzzy-filter projects & sessions; @name msg ⏎ sends straight to it

yn

Answer a permission from the home feed — no terminal needed

↑↓

Drive Claude's option menus mirrored in the card · ⌘↑↓ moves between sessions

⏎

Open the terminal on an empty box — or send your typed reply

⌘+⏎

Open in Ghostty — the same tmux session, in a real window

⌘+[

Step back from the embedded terminal to the home feed

tab

Autocomplete the @token, then keep typing your message

⌘⇧+F

Float ⇄ dock — always-on-top panel or a normal window

⌘+⌫

Kill the session — with a confirmation, because it can't be undone

[4] ROADMAP

Platform plans

Pass starts on macOS because Claude Code, tmux, notifications, hotkeys, and the share extension can be made tight there first. The next platforms are planned as companions, not half-finished ports.

planned

Mobile version

Approve blocked agents from your phone, send short replies, receive completion alerts, and share mobile links, text, screenshots, or files back to a desktop Pass session.

notification actions · share sheet · session picker

planned

Windows support

A Windows build is planned after the macOS workflow is stable. The target is the same supervision model: global summon, session feed, notifications, and terminal-backed agent control.

Windows 11 target · terminal integration research

planned

Steam distribution

Steam support is on the distribution roadmap for users who prefer Steam library updates, install management, and Steam Deck-adjacent workflows once the cross-platform build is ready.

store packaging · auto-updates · release channel
